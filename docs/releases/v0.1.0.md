# Tabulint v0.1.0 — Spreadsheet Change Review and CI

## Highlights

Tabulint v0.1.0 is the first public release of a deterministic Excel workbook change-review
tool. It turns workbook structure, cell, formula, dependency, and policy differences into one
typed result that can be reviewed in the terminal, JSON, offline HTML, Python, a source-checkout
Streamlit page, or GitHub Actions.

The core principle is simple: programs validate. AI may explain existing evidence, but model text
never creates a finding, risk contribution, rule result, or merge-gate decision.

## Installation

Install the published package from PyPI:

~~~bash
python -m pip install tabulint
~~~

To test unreleased changes, install the current public <code>main</code> branch:

~~~bash
python -m pip install "tabulint @ git+https://github.com/huangyikai05/Tabulint.git@main"
~~~

Verify the installation:

~~~bash
tabulint version
tabulint --help
tabulint compare --help
~~~

## Quick example

~~~bash
git clone https://github.com/huangyikai05/Tabulint.git
cd Tabulint
python examples/generate_demo_workbooks.py

tabulint compare examples/generated/before.xlsx examples/generated/after_risky.xlsx \
  --config examples/tabulint.example.yml \
  --json build/risky-review.json \
  --html build/risky-review.html
~~~

The intentionally risky fixture returns:

~~~text
Risk 100/100 (CRITICAL); 5 changed cells; 1 formula overwrites; 4 failed, 1 warning, and 0 errored rules.
~~~

Exit code <code>1</code> is the expected gate decision.

## Main features

- bounded, non-executing <code>.xlsx</code> and <code>.xlsm</code> inspection;
- semantic cell, formula, copied-pattern, and structure differences;
- formula overwrite and formula range-reduction evidence;
- bounded dependency impact analysis with paths and limit state;
- strict YAML business rules;
- explainable, deduplicated risk scoring capped at 100;
- typed JSON and self-contained offline HTML reports;
- Typer CLI and stable exit-code contract;
- local Streamlit interface from a source checkout;
- public top-level Python comparison helper;
- read-only GitHub pull-request workbook review;
- reproducible synthetic safe and risky demos.

## Security

- VBA is detected but never executed.
- Formulas are read as text and never executed or recalculated.
- External workbook links are recorded but never opened or fetched.
- Local interfaces do not upload workbooks or emit telemetry.
- YAML loading is safe and strictly validated.
- Parser and dependency resource use is bounded.
- Offline HTML is autoescaped and has no CDN dependency.
- The release workflow uses PyPI Trusted Publishing preparation and stores no PyPI token.
- The pull-request workbook gate uses a trusted base checkout and read-only permissions.
- External GitHub Actions are pinned to immutable full commit SHAs and tracked by Dependabot.

GitHub Actions report artifacts may contain derived workbook content, including cell addresses,
formulas, before/after values, sheet names, link names, and rule evidence. Treat those artifacts
with the same care as the underlying workbook.

## Known limitations

- Tabulint does not implement Excel's full calculation engine and does not invent cached values.
- Some complex or dynamic formulas receive only reference-level or unsupported-analysis evidence.
- VBA behavior and many advanced Excel features are not fully analyzed.
- ZIP64 and multi-disk packages are outside the current parser security profile.
- External-link detection cannot cover every possible Excel connection mechanism.
- Added/deleted workbooks cannot be semantically paired and fail closed by default in CI.
- Risk scores prioritize review; they are not probabilities, correctness proofs, or professional
  financial-audit conclusions.
